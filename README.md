# ПК «Мигратор ЭПС» — лендинг

Статический маркетинговый лендинг продукта **ПК «Мигратор ЭПС»** (Bezant) — ПО для миграции почты **Microsoft Exchange → CommuniGate Pro**.

Чистый HTML/CSS/JS без фреймворков и сборки. Рассчитан на desktop, корректно рендерится в диапазоне ширины вьюпорта **1200–1920px**.

## Запуск

Сборки нет. Достаточно открыть `index.html` в браузере. Если мешает загрузка ассетов по `file://`, поднимите статику локально:

```bash
npx serve .
# либо
python -m http.server
```

Linter и тесты не настроены.

## Структура

Страница — один `index.html` из 14 секций + header и footer (в порядке документа: Hero, Data, Utp, Speed, Graph, Calculator, Sync, Info, Table, Video, Cost, Form, Faq, footer). CSS и JS разбиты по назначению, а не бандлятся.

```
index.html                 # вся разметка страницы
css/
  tokens.css               # дизайн-токены (цвета, шрифты, отступы) + механизм fluid-масштабирования
  base.css                 # reset + общие примитивы (.btn, .card, .tooltip, .container, ...)
  sections/<name>.css      # по файлу на секцию, подключаются в <head> в порядке документа
js/
  main.js / header.js      # общая логика и хедер
  calculator.js            # логика калькулятора (порт из эталонной реализации)
  video-carousel.js        # карусель видео (3 слота prev/active/next)
  form.js                  # валидация и отправка формы
assets/
  images/<section>/        # экспорты из Figma, по папке на секцию
  icons/                   # SVG-иконки и логотипы
calculator/
  calculator_v2_4_02.html  # эталон-референс калькулятора (регресс-baseline, НЕ менять)
```

## Ключевые особенности

### Fluid-масштабирование (1200–1400px), фикс дальше (1400–1920px)

Вся вёрстка в `rem`, где `1rem = 16px` откалиброван под ширину макета Figma 1440px. Root font-size — fluid только до 1400px, затем замирает:

```css
html { font-size: clamp(13.333px, 1.11111vw, 15.556px); }
```

Между 1200 и 1400px это пропорционально масштабирует все `rem`-размеры без медиазапросов. Выше 1400px root пиннится на 15.556px — ничего дальше не растёт. Контентные ряды, которые должны перестать расширяться, дополнительно несут `max-width: var(--container-width)`.

### Калькулятор (важное ограничение)

`js/calculator.js` дословно портирует логику расчёта из `calculator/calculator_v2_4_02.html` — эталона, который **нельзя изменять**. Скрипты подключаются как обычные `<script src>` (без `type="module"` и `"use strict"`), иначе сломается implicit-global в `calculate()`. Все DOM-id, на которые ссылается скрипт, должны сохраняться в новой разметке.

Проверка изменений UI калькулятора: открыть `calculator/calculator_v2_4_02.html` и `index.html#calculator` рядом, выставить одинаковые значения слайдеров/чекбоксов и сверить итоговый текст — он должен совпадать побайтно.
